Skill：给 Agent 装上「操作手册」
没有 Skill 的 Agent 像新员工摸索，有 Skill 的 Agent 像老手照章执行
用户指令：「帮我发版」— 同一个任务，左右两个 Agent 的执行过程截然不同
低效 没有 Skill 的 Agent
轮次 0
Token 0
错误 0
每一步靠通用能力"摸索"
高效 有 Skill 的 Agent
轮次 0
Token 0
错误 0
按 Skill 手册逐步"执行"
📋
Skill 定义卡
一份 SKILL.md 文件 = 一项技能
技能名称 release
触发条件
发版 打包 release 上线
执行步骤
1跑测试 → 2构建 → 3更新版本 → 4打 Tag → 5推送远端
允许工具
run_command edit_file git
存储路径：~/.alice/skills/release/SKILL.md 或 {项目}/.alice/skills/
📌 Skill 本质是"经验封装"——把高级工程师的操作流程写成 Agent 能理解的手册。Agent 不再每次从零摸索，而是站在前人肩膀上执行。
📌 效率差距可达 3 倍以上——没有 Skill 时 Agent 靠通用能力"摸索"，有 Skill 时按最佳实践"执行"。Token 消耗差 3.75 倍，错误从 3 个变 0 个。
📌 产品经理要回答的问题——为哪些高频操作预装 Skill？允许用户自建 Skill 吗？Skill 之间冲突了怎么办？谁来维护和更新 Skill？
Takeaway Skill 把「摸索」变成「执行」——同一个发版任务，没有 Skill 时 Agent 试错 12 轮花 1.5 万 Token 还出 3 个错；有 Skill 时 5 轮搞定只花 4000 Token 零错误。Skill 是 Agent 的经验传承机制。
